**Fence deadlock fix (VulkanBootstrapAppRender.cpp):**
In `drawFrame()`, move the `vkResetFences(device_, 1, &inFlightFence_)` call (currently at line 484, before the vertex memcpy) to AFTER successful image acquisition and BEFORE `vkQueueSubmit()`. The fence must only be reset when we are certain a queue submit will happen to re-signal it. The corrected ordering is:
1. `vkWaitForFences()` (existing, line 452)
2. `vkAcquireNextImageKHR()` (existing, lines 455-461)
3. If `VK_ERROR_OUT_OF_DATE_KHR`, call `recreateSwapchain()` and return -- fence was NOT reset, still signaled, safe
4. Copy vertex data to mapped buffer (existing, lines 472-482)
5. `vkResetFences(device_, 1, &inFlightFence_)` -- MOVED HERE (was line 484)
6. `vkResetCommandBuffer()` (existing, line 485)
7. `recordCommandBuffer()` (existing, line 486)
8. `vkQueueSubmit()` with `inFlightFence_` (existing, line 502)

Do NOT change any other logic in drawFrame(). This is a one-line move.

**Validation layer enablement (VulkanBootstrapAppCore.cpp):**
In `createInstance()`, add validation layer support guarded by `MCE_VULKAN_VALIDATION` preprocessor define:
1. Before `vkCreateInstance()`, add:
```cpp
#ifdef MCE_VULKAN_VALIDATION
const char* validationLayers[] = {"VK_LAYER_KHRONOS_validation"};
createInfo.enabledLayerCount = 1;
createInfo.ppEnabledLayerNames = validationLayers;
std::cout << "[mce_vulkan_boot] Vulkan validation layers enabled\n";
#endif
```
2. Also add `VK_EXT_debug_utils` to the instance extensions when validation is enabled. In `getRequiredInstanceExtensions()`, add:
```cpp
#ifdef MCE_VULKAN_VALIDATION
extensions.push_back(VK_EXT_DEBUG_UTILS_EXTENSION_NAME);
#endif
```
